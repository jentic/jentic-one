# Docs-Drift Job ‚Äî Steering Notes

Context and recurring drift hotspots for the documentation-drift cron job.
Additive to `CLAUDE.md` / `AGENTS.md`; does not replace them.

## How to detect drift in this repo

Docs are checked by hand-diffing prose against the actual code/assets. There is
no automated docs-drift linter yet. The reliable cross-references are:

- **Make targets** ‚Äî the source of truth is `Makefile`. Docs that list or
  describe `make <target>` (CLAUDE.md Commands table + Quick Start, README
  Development table + Quick start blocks, CONTRIBUTING, deploy/README) drift
  when targets are renamed/removed. Verify with:
  `grep -oE '^[a-z][a-z-]+:' Makefile`.
  - `make check` runs **`lint score detect-secrets test-arch`** ‚Äî it does NOT
    run unit tests. Several docs historically claimed "unit + arch tests".
  - `make install` runs `sync ui-setup hooks` (includes UI deps).
- **`tools.deploy` CLI** ‚Äî the local k8s workflow is `uv run python -m
  tools.deploy <cmd>` (Click). An older `make deploy-local MODE=...` target was
  removed; deploy/README still had stale references to it. Commands live in
  `tools/deploy/cli.py` (`grep '@cli.command'`).
- **Project Layout (CLAUDE.md)** ‚Äî compare each listed path against `ls`. The
  Go **`cli/`** module (two binaries: `jenticctl`, `jentic`) is a top-level
  component that CLAUDE.md's layout omitted for a while. `shared/` also grows
  new files/dirs (e.g. `telemetry/`, `lookups.py`, `provider_config_store.py`,
  `db/backends/`).
- **Config / DB backend** ‚Äî the DB layer supports **two backends** selected per
  database via `DatabaseConfig.backend`: `postgres` (default,
  `postgresql+asyncpg`) and `sqlite` (`sqlite+aiosqlite`). Schema
  (`src/jentic_one/shared/config.py::DatabaseConfig`) and
  `config/local-sqlite.yaml` are the source of truth. There is no bare `Base` ‚Äî
  ORM bases are `RegistryBase` / `ControlBase` / `AdminBase`.
- **Surfaces** ‚Äî five surfaces: registry, control, admin, auth, broker. Older
  docs said "modules (broker, control)".
- **Generated reference** (`docs/reference/`) is regenerated by `make endpoints`
  and guarded by CI drift tests ‚Äî do NOT hand-edit `endpoints.{md,json}`.

## Files that tend to drift

- `CLAUDE.md` ‚Äî Commands table, Project Layout.
- `README.md` ‚Äî Development & testing table, Quick start blocks.
- `CONTRIBUTING.md` ‚Äî `make check` description.
- `docs/context-and-config.md` ‚Äî DB backend / bases / surfaces list.
- `config/README.md` ‚Äî Files table (must list all `config/*.yaml`).
- `deploy/README.md` ‚Äî deploy CLI invocation vs. old Make targets.

## Not drift (verified current ‚Äî don't "fix")

  registry/control/admin schemas in `init-schemas.sql`).
- deploy/README Bitnami PostgreSQL note ‚Äî accurate (chart pins
  `bitnamilegacy/postgresql`).
